Abstract¶

The GLUE specification is an information model for Grid entities described using the natural language and UML Class Diagrams. As a conceptual model, it is designed to be independent from the concrete data models adopted for its implementation. This document defines the GLUE 2.1 model by extending the GLUE 2.0 model to include Cloud resources and services. Rendering to concrete data models such XML Schema, LDAP Schema and SQL are provided in a separate document.

Hi there,

before I start, I should say that I generally like the cloud extension, which Shiraz asked me to review. I do have some issues understanding parts of the model though, which may be owed to my ignorance of the general GLUE approach.

Best regards,
Björn

The current status of the document, in particular the cloud computing section,
  needs a language overhaul. Some portions of the text are incomprehensible. In
  particular, this seems to be true for the description column in the entity
  tables.
With regards to content, there are a number of issues that should be solved
  before publishing this document. In particular, these are related to the
  accelerators.
Regarding the Accelerators, I am not sure I fully understand the meaning of
  the various ones. Also, I would have expected there to be a relation among
  them, e.g. the description of CloudComputingVirtualAccelerator clearly states
  "A cloud computing instance may be associated with one or more virtual
  accelerators." However, there is a special CloudComputingInstanceAccelerator
  that can be directly linked to an instance. At the same time, the relation of
  CloudComputingVirtualAccelerator and CloudComputingInstanceAccelerator are not
  clear.
All Accelerators are described as "information about the usage level of the
  virtual accelerator". I do not understand what you mean by usage level.
Some general remarks about the language:
Turn "who" into "which" when you refer to things rather than people.
Wording: "With virtual CPU it is intended ..." -> "Virtual CPU is the count of
  cores as seen by the VM.
Section 8.5: "... layer may be not exposed ..." -> "... layer may not be
  exposed ..."
8.5 -> "TotalCPUs": "... total number of physical CPU cores ...", i.e. omit
  the s from CPUs.
-> "InstanceMaxRAM": "phisical" -> "physical"
-> "CloudComputingImage.ID": "... one or more cloud computing image_s_"
-> "Benchmark.ID": "... . These benchmarks ..."
8.13: what is a price voice?
Would it help if I return the DOCX with change tracking and comments?
There are probably other issues, too, but I have little time to work on them.
