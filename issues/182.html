fillByte - without Byte Value entities (optional feature) one might not be able to specify a byte.

Added by Steve Hanson about 8 years ago. Updated about 8 years ago.

Status:closed Start date:09/26/2013
Priority:Normal Due date:
Assignee:Michael Beckerle % Done:

100%
Category:-
Target version:-
Document Type:Proposed Recommendation

Description

fillByte - if Byte Value Entities (aka rawbytes) feature is not implemented, and encoding is not SBCS then there is no way to specify a fillByte that is a byte.

Even with SBCS, there will be values one may not be able to create using characters.

This is just an observation, but it might be an undesirable restriction.
